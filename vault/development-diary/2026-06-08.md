# 2026-06-08 — Erit World Development Diary

## Main Purpose

Today focused on testing the Erit / Vys layer method directly in Twine.

The goal was to see whether the worldbuilding rule could become a playable structure instead of remaining only a written method.

## Main Discovery

The layers should not behave like simple reputation bars.

They work better as scales of influence:

- Layer 1 — City / World State
- Layer 2 — Faction Pressure
- Layer 3 — NPC / Local Behavior

Layer 2 and Layer 3 can affect each other's strength, but they keep separate identities.

A faction remains itself. A local action remains local. What changes is how strongly each action affects the current condition of the city.

## Layer Rule

The working rule became:

```text
Actor decides layer.
Pressure decides city.
City decides future context.
```

This made the system easier to read.

A worker action belongs to local behavior.

A faction operation belongs to faction pressure.

The city state changes only after those pressures accumulate.

Because each quest only needs to operate within its assigned layer, quest design became easier as well. Individual quests no longer need to account for every possible world outcome. They only need to create pressure inside their natural scope, while the layer system handles how those effects spread into larger city changes.

## Twine Test

The Twine prototype keeps quest passages separate from the city-state calculation.

Quest passages set layer values.

The pressure passage collects those values and sends the result to the city-state check.

This keeps individual quests simple while still allowing systemic consequences.

The separation also made it easier to build new quests. Local stories can focus on NPC behavior, faction stories can focus on organizational goals, and both automatically contribute to the broader simulation without requiring custom logic for every scenario.

## Design Takeaway

The clearest rule became:

```text
Experiences create pressure.
Influence decides how far that pressure spreads.
Layers decide where that pressure acts.
```

This keeps the system flexible without reducing it to a simple good / bad score.

This also made quest design easier.

Each quest can stay inside its natural scope: local behavior, faction pressure, or city state. That creates useful constraints without breaking immersion, because the quest does not need to force a world reaction by itself. It creates pressure, and the layer system decides how far that pressure spreads.
