# NPC System Relationship Model

## Purpose

Defines how an NPC relates to a large belief system or social order — Regulatus in Erit's world — rather than defining what the system is. Working vocabulary, not final categories: the labels are meant to change if they ever stop helping or start flattening characters.

## Core Principle

NPCs should not all respond to a civilization-wide system the same way. The useful question isn't "does this NPC follow Regulatus?" — it's "what relationship does this NPC have with the system?"

## Working Relationship Types

**Believer.** Accepts the public meaning sincerely — doesn't think they're controlled. "Order protects us. The system is not perfect, but chaos is worse." Should sound reasonable from inside their inherited worldview, not stupid. Defends hierarchy sincerely, sees records as protection, treats obedience as maturity.

**Rejector.** Sees the system as control, brainwashing, or theft — rebels, exiles, forbidden historians, citizens pushed too far. "They taught us pride so we would not notice the chains." Not automatically good — rejecting the system doesn't make someone kind, honest, or safe.

**Damaged Believer.** Still speaks the system's language but personal experience has cracked their faith. "Regulatus protected my family once. Regulatus also abandoned my child." Lives in contradiction rather than changing from belief to rebellion instantly — repeats official phrases, then hesitates; bends rules quietly while defending the system publicly.

**Outsider.** Doesn't fully share the system's emotional language — foreign, isolated, newly displaced, raised outside its influence. May see clearly what insiders normalize, or misread why citizens act the way they do. Not automatically wiser — may miss context or underestimate danger.

**Performer/Manipulator.** Knows the public meaning is constructed, plays along because the performance maintains power. House Ventari is the strongest example: publicly, "Regulatus protects the whole"; privately, "Regulatus keeps people proud enough not to ask who benefits." Not automatically a cartoon villain — some believe the lie is necessary, some fear collapse without it, some simply enjoy the power.

## Core NPC Stack

A future NPC builds through layered pressure: System Relationship → Location Pressure → Group Pressure → Personal State. These layers follow the same deeper worldbuilding logic — experience → emotion → reason → behavior → atmosphere → quest logic — applied fractally: the world uses the method, cities use the method, groups use the method, NPCs use the method.

## Design Rule

Do not use these labels as boxes that replace writing. Use them as pressure tests. If an NPC feels flat, ask what system relationship shaped them, what location and group pressure bend their behavior, and what personal state makes them specific. If the answer can't be found, the NPC may not yet be grounded enough.

This model can fail if it becomes too rigid — the goal isn't classifying every person perfectly, it's making behavior feel like it comes from the world instead of quest convenience.
